# Security policy

## Supported versions

Security fixes are applied to the latest release on the `main` branch.

## Reporting a vulnerability

Do not open a public issue for vulnerabilities that expose data, bypass enforcement, or create a denial of service. Use GitHub's private vulnerability reporting feature in the repository Security tab.

Include the affected version, reproduction steps using synthetic data, expected impact, and a proposed mitigation if available. Never include live credentials, personal data, confidential prompts, or active malware.

This project is a defensive reference implementation. A missed detection by itself is not considered proof of a vulnerability unless it violates a documented security boundary.
